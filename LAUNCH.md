## 1. Redeploy в Coolify

1. Открой панель Coolify (адрес твоей панели на сервере Beget), залогинься.
2. Слева **Projects** → проект с приложением → приложение (X-Raya / sourcing-agent).
3. Справа сверху кнопка **Redeploy** (или **Deploy**). Нажми, дождись в логах строки об успешном старте и зелёного статуса **Running**.
4. Открой x-raya.space и обнови со сбросом кэша: **Cmd+Shift+R**.
5. Быстрая проверка, что приехала свежая версия: в тулбаре результатов должна быть кнопка **«Проверить выдачу»**, внизу страницы — блок **«Peleng Pro // ранний доступ»**.

## 2. Volume на папку data (чтобы заявки не стирались)

Зачем: сервер пишет email-заявки в `data/leads.jsonl` внутри контейнера. Без volume файл живёт до первого редеплоя.

1. В Coolify открой приложение → вкладка **Storages** (может называться **Persistent Storage**).
2. **+ Add**: 
   - **Destination Path** (путь в контейнере): `/app/data`
   - Name/Source — любое, например `peleng-data` (Coolify создаст сам).
3. **Save** → **Redeploy**.
4. Проверка: оставь тестовую заявку на сайте → в Coolify открой вкладку **Terminal** приложения → `cat /app/data/leads.jsonl` — должна быть строка с email. Сделай ещё один Redeploy → повтори `cat` — строка должна остаться.

## 3. Ключ Serper.dev (проверка выдачи) — простой путь

Рекомендую Serper вместо Google CSE: регистрация в 2 клика, ключ сразу, 2500 бесплатных проверок, без заглушек и капризных тумблеров.

1. Зайди на **serper.dev** → **Sign up** (можно через Google-аккаунт).
2. После входа открой раздел **API Key** (в дашборде) → скопируй ключ.
3. Coolify → приложение → **Environment Variables** → добавь `SERPER_KEY` = скопированный ключ → **Save** → **Redeploy**.
4. Проверка: поиск на сайте → кнопка **«Проверить выдачу»** → на карточках бейджи: **«выдача есть»** / **«мало · N»** / **«пусто в Google»** (пустые карточки тускнеют).

Как читать бейджи (Serper смотрит первую страницу Google):
- **пусто** — результатов нет, канал для этого запроса бесполезен;
- **мало · N** — на первой странице всего N результатов (1–9), выдача тонкая;
- **выдача есть** — полная первая страница, кандидатов достаточно.

**Запасной вариант — Google CSE** (если не хочешь Serper): 100 проверок/день бесплатно, но флоу капризный.
1. programmablesearchengine.google.com → создать поисковик (в поле сайта впиши заглушку `www.google.com`, без неё не даёт создать).
2. Открыть его → **Настройки → Основные** → включить тумблер **«Искать по всему интернету»** (становится активным только когда в списке есть хотя бы один сайт — заглушку не удаляй). Скопировать **Search engine ID** = `GOOGLE_CSE_CX`.
3. console.cloud.google.com → New Project → **APIs & Services → Library → Custom Search API → Enable** → **Credentials → Create → API key** = `GOOGLE_CSE_KEY`.
4. Coolify → Environment Variables: `GOOGLE_CSE_KEY` и `GOOGLE_CSE_CX` → Redeploy. (Если задан `SERPER_KEY`, он приоритетнее.)

## 6. Домен

**Выбор:** для RU-аудитории рекомендую **peleng.ru** (~200–900 ₽/год, reg.ru / timeweb, нужны паспортные данные) — быстрее и дешевле всего. `.io/.space` — только если планируешь международность (нужна зарубежная карта).

1. Проверь свободность и купи домен у регистратора.
2. Узнай IP своего сервера: `ping x-raya.space` (или посмотри в Beget).
3. В DNS-панели регистратора создай записи:
   - `A` запись: имя `@`, значение — IP сервера;
   - `CNAME`: имя `www`, значение — `peleng.ru.` (или тоже A на IP).
4. Подожди 10–60 минут (распространение DNS): `ping peleng.ru` должен показать IP сервера.
5. Coolify → приложение → **Domains** → добавь `https://peleng.ru` → Save. Coolify сам выпустит Let's Encrypt-сертификат (DNS уже должен указывать на сервер).
6. Старый x-raya.space оставь как второй домен на время переезда.

## 7. Telegram-уведомления о заявках

1. В Telegram открой **@BotFather** → `/newbot` → имя `Peleng Leads` → username вида `peleng_leads_bot` → скопируй **token** (вида `123456:AA...`).
2. Напиши своему новому боту любое сообщение (`/start`).
3. Открой в браузере: `https://api.telegram.org/bot<ТОКЕН>/getUpdates` → найди `"chat":{"id":123456789` — это твой **chat_id**.
4. Coolify → Environment Variables: `TG_BOT_TOKEN` и `TG_CHAT_ID` → Save → Redeploy.
5. Тест: оставь заявку на сайте → боту должно прийти «🎯 Peleng: новая заявка на Pro».

## 8. Тест движка с Егором

1. Собери 20–30 **реальных формулировок** запросов — как пишут рекрутеры в заявках/вакансиях (не выдуманных). Источники: ваши прошлые заявки, hh-вакансии, чаты рекрутеров.
2. Прогоните каждый через Peleng, заполняя ту же таблицу, что в п.4.
3. Егор дополнительно сверяет **правила каждого канала** со своим документом источников: правильный ли синтаксис, не устарели ли операторы, чего не хватает.
4. Итог (таблица + замечания Егора) — мне: обновлю словарь ролей (xlsx → компилятор) и правила каналов.

## 9. Реферальные ссылки партнёрам

1. Формат: `https://x-raya.space/?ref=egor` (латиницей, без пробелов; после переезда — на новом домене).
2. Раздай каждому партнёру его метку: `?ref=egor`, `?ref=имя2`…
3. Метка автоматически пишется в заявку при оставлении email.
4. Смотреть статистику: Coolify → Terminal → `cat /app/data/leads.jsonl` — каждая строка: email, дата, ref. Подсчёт по партнёру: `grep -c '"ref":"egor"' /app/data/leads.jsonl`.

## 10. Договорённости с партнёрами письменно

Минимум — одна страница, зафиксированная хотя бы в email/мессенджере с явным «согласен» от каждого:

1. Перечень участников и ролей.
2. Реферальная схема: % от платежей приведённых пользователей (20–30%), срок — 12 мес с регистрации пользователя.
3. Доля партнёра-логики: %, условие активного участия, пересмотр раз в 6 мес.
4. **Sunset**: доля/начисления прекращаются после 3 мес без вклада.
5. **IP** (код, словарь, бренд) принадлежит тебе.
6. Продуктовые и ценовые решения — за тобой.
7. Для приёма денег понадобится статус: **самозанятость** (до 2.4 млн ₽/год, оформляется в приложении «Мой налог» за 10 минут) или ИП. Лучше решить до включения оплат.

Могу сгенерить драфт соглашения — попроси.

## 12. SignalHire триал (после беты)

1. Зарегистрируйся на signalhire.com (есть бесплатные кредиты на старте).
2. Собери 20 ссылок на **реальных кандидатов** (LinkedIn/VK/GitHub профили из ваших прошлых поисков).
3. Раскрой контакты по каждому, посчитай hit-rate: у скольких получен валидный телефон / личная почта.
4. **>60%** — интегрируем (сначала BYOK: пользователь вставляет свой ключ). **<50%** — тестируем альтернативы (X-Ray Contact и др.).
5. Параллельно спроси агентства-партнёров: у кого уже есть подписка SignalHire (это готовые BYOK-пользователи).
