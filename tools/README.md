# Словарь ролей X-Raya — как вести

Источник правды — **`../X-Raya-roles.xlsx`** (лист «Роли»). Правишь таблицу → компилируешь → вставляешь в `index.html`.

## Колонки таблицы
| Колонка | Смысл |
|---|---|
| Ключ | уникальный id роли (латиницей, без пробелов). Не менять у существующих без нужды. |
| Категория | `it` или `nonit` |
| Флаги | `exec` — показывает площадки руководителей (Crunchbase, E-xecutive, 4CIO) |
| Профильные площадки | `kaggle`, `behance`, `dribbble`, `dprofile`, `logist`, `hf` (Hugging Face) — через ` \| ` |
| GitHub-ключ | англ. слово для нативного поиска GitHub (только для IT) |
| Русское название | как показывать роль |
| Стек по умолчанию | подставляется, если в запросе стек не указан (через ` \| `) |
| Синонимы для запросов | идут в поисковые строки (LinkedIn, hh и т.д.), через ` \| ` |
| Алиасы распознавания | по ним движок узнаёт роль во вводе: EN + RU + аббревиатуры, через ` \| ` |

Списки внутри ячейки разделяются ` | ` (пробел-палка-пробел).

## Как движок выбирает роль
1. Берётся **самое специфичное (длинное) совпадение** алиаса, а не первое по списку.
2. Транслитерация RU→EN бриджится автоматически (`дата аналитик` = `data analyst`), карта в `index.html` → `LOAN`.
3. Роль `Developer` — generic-fallback: срабатывает, только если ничего конкретнее не нашлось. Не удаляй её.

## Рабочий цикл
```bash
cd tools
npm i                 # один раз (ставит xlsx)
# ... правишь ../X-Raya-roles.xlsx ...
node compile.js       # xlsx → tools/titles.generated.js
node test-roles.js    # прогон распознавания; при провале — код 1
```
Затем скопировать массив из `titles.generated.js` в `index.html` вместо блока `const TITLES = [ ... ];`.

Первый раз таблица собрана из `roles-data.js` командой `node build-xlsx.js` (нужен только для пересоздания с нуля).

## ⚠️ При вмёрдже в index.html — одна правка движка
Раньше Hugging Face включался по `if(p.role.key==='Data')`. В новом словаре роль Data разбита
(DataScientist / DataEngineer / DataAnalyst / MLEngineer / BIAnalyst). Триггер теперь по тегу `hf`:

```js
// было:
if(p.role.key==='Data'){ ... Hugging Face ... }
// стало:
if((p.role.extra||[]).includes('hf')){ ... Hugging Face ... }
```
Остальная логика build() совместима (kaggle/behance/dribbble/dprofile/logist уже по `extra`).

## Тесты
`test-roles.js` гоняет движок из `index.html` поверх `titles.generated.js`. Добавляй туда кейсы
`['запрос','ОжидаемыйКлюч']` при расширении словаря — это ловит регрессии.

## Покрытие Candy

`node candy-gap.js` — прогоняет все 192 специализации Candy (`candy-specializations.md`) через движок
и показывает, что распознаётся, а что падает в generic. Сейчас покрыто **187/192**.
Осознанно НЕ роли (остаются generic): `Haskell` (язык → Developer + стек), `НДО`,
`Разработка TWork процедур`, `Методолог IDM`, `IT Other` — внутренний жаргон Candy без
публичного аналога для сорсинга.
