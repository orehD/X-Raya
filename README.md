# X-Raya — сорсинг-агент

Веб-приложение: по нативному запросу рекрутера генерит boolean / X-Ray ссылки для поиска
кандидатов по hh.ru, Habr Career, LinkedIn, GitHub, Telegram, конференциям/митапам,
Stack Overflow, VK, GitLab и профильным площадкам.

Работает **полностью в браузере**, без бэкенда, ключей и сборки — один файл `index.html`.

## Запуск локально

Просто открой `index.html` двойным кликом. Или подними локальный сервер:
```bash
cd sourcing-agent
python3 -m http.server 8080
# открой http://localhost:8080
```

## Деплой в интернет (выбери любой, все бесплатные)

**1. Netlify Drop (самый быстрый, 30 секунд):**
- Зайди на https://app.netlify.com/drop
- Перетащи папку `sourcing-agent` в окно → получишь публичную ссылку.

**2. Vercel:**
```bash
npm i -g vercel
cd sourcing-agent && vercel
```

**3. GitHub Pages:**
- Создай репозиторий, залей `index.html`.
- Settings → Pages → Source: ветка `main`, папка `/root` → сайт на `https://<user>.github.io/<repo>/`.

**4. Cloudflare Pages** — подключи репозиторий, build command пустой, output dir `/`.

## Как устроено

- `parse()` — разбирает запрос: роль, грейд, город, технологии (словари RU+EN зашиты в `index.html`).
- `build()` — собирает строки и URL под синтаксис каждого источника.
- Каждая карточка показывает готовую boolean-строку + кнопки «Открыть» и «Копировать».

## Что легко докрутить

- **Новые города** — добавь объект в массив `CITIES` (нужны: `hh` area-id и `habr` code).
- **Новые технологии / синонимы** — массив `TECH` (флаг `lang:1` для языков программирования).
- **Новые роли** — массив `ROLES` (`detect` — триггеры, `syn` — синонимы названия, `extra` — доп.площадки).
- **Новые источники** — добавь карточку в `build()` с группой `grp`.

## AI-режим (сервер хранит ключ, пользователи ничего не настраивают)

Фронт зовёт серверный прокси `/api/ai`, который хранит API-ключ как переменную окружения.
Пользователи видят только галочку «AI-интерпретация» и кнопки «шире / уже / по-другому» —
ключ и модель скрыты.

- `api/ai.js` — serverless-функция (Anthropic). Хранит ключ, проксирует запрос.
- Фронт: `AI_ENDPOINT = '/api/ai'` (можно переопределить `window.XRAYA_AI_ENDPOINT`).

### Полный деплой через GitHub + Vercel (рекомендую)

1. Залей папку в репозиторий:
   ```bash
   cd sourcing-agent
   git init && git add . && git commit -m "X-Raya"
   gh repo create x-raya --public --source=. --push
   ```
2. На https://vercel.com → **Add New Project** → импортируй репозиторий x-raya (Framework: Other,
   ничего настраивать не нужно — Vercel сам отдаст `index.html` и поднимет `api/ai.js`).
3. Project → **Settings → Environment Variables** добавь:
   - `ANTHROPIC_API_KEY` = твой ключ Anthropic (обязательно)
   - `AI_MODEL` = `claude-haiku-4-5-20251001` (необязательно; можно `claude-sonnet-4-6` для качества)
4. **Deploy**. Готово — публичная ссылка вида `https://x-raya.vercel.app`, AI уже работает у всех,
   ключ нигде не виден. Каждый `git push` обновляет сайт автоматически.

### Cloudflare Pages — альтернатива
Аналогично: подключи репозиторий, добавь функцию в `functions/api/ai.js` (формат Pages Functions
чуть отличается — скажи, дам версию под Cloudflare) и переменную `ANTHROPIC_API_KEY`.

### Свой / внутренний прокси
Если хочешь крутить через внутреннюю модель — поменяй в `api/ai.js` вызов на свой endpoint
(формат тела можно оставить тем же: `{system, user}` → `{text}`).

> Без задеплоенного прокси AI-кнопки и галочка просто отдадут ошибку в тосте — детерминированный
> движок при этом продолжает работать офлайн.
